Privacy Policy

Privacy Policy

Effective Date: April 24, 2026
LanTrack (“the App”, “we”, “our”, or “us”) is a privacy-first, offline OJT (On-the-Job Training) tracking application designed to give users full control over their data. This Privacy Policy explains how information is handled within the application.

1. Overview

LanTrack is built with a strict offline-first architecture. This means:

  • Your data is stored locally on your device
  • We do not require internet access for core functionality
  • We do not collect personal data on external servers
  • We do not sell or share user data with third parties
  • Your information remains entirely under your control.

2. Information We Collect

LanTrack does not collect personally identifiable information externally. However, the app stores the following locally on your device:

2.1 User-Generated Data

  • Time tracking sessions (check-in / check-out logs)
  • Journal entries and reflections
  • Task lists and productivity data
  • Schedule configurations
  • Automation settings

2.2 System Data

  • App preferences (theme, settings, seed color)
  • Automation logs
  • Backup files generated by the user

3. Data Storage

All data is stored using:

  • Room Database (SQLite)
  • Local device storage (for backups and exports)

No cloud storage is used unless explicitly added by the user through external tools.

Zero external transmission: Your check-ins, journals, and tasks never leave your device unless you manually export them.

4. Data Usage

The data stored in LanTrack is used strictly for:

  • Tracking work and study hours
  • Managing productivity tasks
  • Supporting journaling and reflection
  • Powering automation features (optional)
  • Generating local reports and summaries

We do not analyze or transmit your data externally.

5. Automation & Background Processing

LanTrack may perform automated actions such as:

  • Suggesting check-ins or check-outs
  • Logging scheduled events

These actions:

  • Are executed locally on the device
  • Are fully configurable in Settings
  • Can be disabled at any time

All automation events are recorded in a local audit log for transparency.

6. Backup and Export

Users may create manual backups of their data.

Backup Behavior:

  • All data is exported into a single JSON file
  • Stored locally on the device
  • No automatic upload to external servers

Restore Behavior:

  • Existing data is cleared before restoring backup
  • Data is fully reconstructed from the JSON file

7. Data Sharing

LanTrack does not: Collect user data remotely · Sell user data · Share user data with advertisers · Send analytics to external servers. Any data sharing occurs only if the user manually exports and shares files.

8. Permissions

LanTrack may request the following permissions:

  • Storage Access: For exporting/importing backups
  • Notifications: For reminders and automation alerts
  • Foreground Service (if enabled): For time tracking consistency

No permission is used to collect personal data externally.

9. Third-Party Services

LanTrack does not integrate third-party analytics, advertising SDKs, or tracking services. If future integrations are added, this policy will be updated and clearly disclosed.

No Firebase, no Crashlytics, no telemetry.

10. Data Security

We implement local security practices including:

  • Encrypted or sandboxed local storage (based on Android OS security model)
  • No external network transmission of user data
  • Optional device-level protection (PIN, biometrics if enabled by user device)

However, users are responsible for securing their own devices.

11. Children’s Privacy

LanTrack does not knowingly collect data from children under 13. Since all data is stored locally, responsibility remains with the device user or guardian.

12. User Rights

Users have full control over their data, including:

  • View all stored data
  • Edit or delete sessions, journals, and tasks
  • Export all data
  • Wipe all data at any time

13. Changes to This Policy

We may update this Privacy Policy to reflect new features, security improvements, or legal requirements. All changes will be reflected within the app and updated with a new effective date.
